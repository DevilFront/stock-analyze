## Action Plan: 데이터 수집 · 벡터 DB · 프론트엔드 시각화

### 1. 뉴스/공시 수집기 (Scraper / API)

- 한국 시장 기준으로 다음 API를 연동해 기업 이벤트 데이터를 꾸준히 쌓는다.
  - **OpenDART API**: 공시(전자공시) 데이터 수집
    - 주요 항목: 정기/수시 공시, 실적 공시, 증자/감자, 배당, 채권 발행 등
    - 필터링: 종목 코드별, 공시 유형별, 기간별
  - **Naver / Google News API** 또는 자체 스크래퍼: 뉴스 데이터 수집
    - 주요 항목: 제목, 요약, 발행일, 언론사, 원문 URL
    - 키워드: 기업명 + 티커, 주요 제품/서비스명, 섹터 키워드
- 수집 파이프라인 개념
  - 정기 배치(예: 5~15분 단위) 또는 웹훅 기반으로 새 공시/뉴스를 가져온다.
  - 가져온 원문/요약 텍스트를 **전처리 + 요약 + 심각도 분류**까지 한 번에 수행한다.

### 2. 벡터 DB 연동 (선택사항이지만 유료급 기능의 핵심)

- 목적: 기업별 과거 리포트/공시/뉴스를 벡터 DB에 쌓아두고,
  - "이번 공시가 지난번 공시와 비교해서 어떤 점이 달라졌는가?"
  - "이 기업의 최근 1년 리포트에서 반복 등장하는 키워드는 무엇인가?"
  - 같은 질문에 바로 답할 수 있게 한다.
- 기본 설계
  - 엔티티: `기업ID`, `데이터 타입(리포트/공시/뉴스)`, `시점(날짜)`, `텍스트`, `메타데이터(EPS 영향도, 호재/악재/중립 등)`
  - 벡터화: OpenAI Embeddings 또는 타 모델로 텍스트를 임베딩 → 벡터 DB에 저장
  - 질의: "현재 공시와 가장 유사한 과거 공시 3개" / "과거의 비슷한 이벤트와 이번 이벤트의 차이점" 등을 검색해 AI 프롬프트에 주입
- 이 벡터 DB를 활용하면, 리포트에서 다음과 같은 문장이 가능해진다.
  - "이번 배당 정책은 3년 평균 대비 배당 성향이 X%p 높아졌으며, 과거 공시와 비교 시 **주주환원 강화 기조가 뚜렷**하게 나타납니다."

### 3. 프론트엔드 시각화 (Bar / Area Chart)

- 목표: 텍스트만 나열하는 대신, AI 리포트 중간중간에 **재무제표와 이벤트의 흐름을 시각화**해 가독성을 극대화한다.
- 주요 차트 예시
  - **Bar Chart**
    - 최근 4~8개 분기의 매출 / 영업이익 / 순이익 막대 그래프
    - 영업이익률(%) 막대를 별도 축으로 표시해 마진 변화를 직관적으로 보여줌
  - **Area Chart**
    - 매출 또는 순이익의 장기 추세(3~5년)를 면적 그래프로 표현
    - 특정 공시/뉴스 발생 시점을 수직선 또는 마커로 표시해 **이벤트 전후 흐름**을 시각적으로 연결
  - **리스크 히트맵(선택)**
    - 금리/환율/업황/규제/경쟁 등 리스크 항목을 축으로 한 간단한 히트맵 또는 막대 그래프
- 구현 포인트
  - Next.js + Tailwind + 차트 라이브러리(예: Recharts, ApexCharts 등)를 활용한다.
  - AI 리포트 컴포넌트에서, 특정 섹션(예: "사업 현황 분석")에 대응하는 차트 블록을 함께 렌더링하여,
    - "텍스트 요약 + 수치 그래프"가 항상 페어로 보이게 만든다.
  - 향후에는 AI가 "이 구간은 차트로 보여주는 게 좋다"고 판단해 **차트 추천**까지 할 수 있도록 확장 가능하다.
